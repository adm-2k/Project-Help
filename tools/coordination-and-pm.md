# Coordinating the Team

_Last verified: 2026-06. Tool prices and free tiers change often, so re-check the linked sites and update this page each cohort (about quarterly)._

Your team needs one place to track who is doing what. This playbook already hands you that place: a markdown [task board](../templates/task-board.md) and an [issue log](../templates/ISSUE-LOG.md) you can use today, with no setup. If you later want something richer, here is how the popular tools compare.

| Tool | Cost | Best for | Works with GitHub? |
|---|---|---|---|
| GitHub Issues + Projects | Free | Tracking work right next to your code, with links to *[pull requests](../GLOSSARY.md)* | Yes — it is GitHub |
| Trello | Free tier | Simple drag-and-drop boards | No — separate from your code |
| Notion | Free for personal use | Flexible docs plus light databases | No — can sprawl if undisciplined |
| Linear | Free tier | Polished issue tracking aimed at software teams | Yes, but more than most first MVPs need |

## Where to start

Start with the playbook's own [task board](../templates/task-board.md) and [issue log](../templates/ISSUE-LOG.md). They are free, need no accounts, and keep your team moving from day one.

When the paper board starts to feel cramped, graduate to GitHub *[Issues](../GLOSSARY.md)* plus a Projects board. The [Break into tasks guide](../guides/step-3-break-into-tasks.md) walks you through it. The win is that your coordination then lives with your code, so the place you plan work and the place you do work are the same.

> Going further on GitHub: Issues can be grouped into a Projects board and linked to pull requests, so the board updates itself as work moves from to-do to in progress to done.
