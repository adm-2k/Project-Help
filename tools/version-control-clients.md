# Ways to Manage Your Code Versions

_Last verified: 2026-06. Tool prices and free tiers change often, so re-check the linked sites and update this page each cohort (about quarterly)._

Once your project lives on [GitHub](../GLOSSARY.md), you need some way to make changes and send them back. There are several, and the common ones are all free. The one you have probably heard of is GitHub Desktop, but it is not the only choice, and you may not even need it.

Here are the common ways teams do this.

| Tool | Cost | Ease for non-coders | Where it runs | Best for |
| --- | --- | --- | --- | --- |
| GitHub Desktop | Free | Highest | Desktop app | Non-coders who want a visual, button-based way to *clone*, *branch*, *commit*, and *push* |
| VS Code Source Control | Free | Medium | Desktop code editor | People already editing in VS Code |
| GitHub web editor (press the `.` key in any repo, or go to `github.dev`) | Free | High | Browser, nothing to install | Small edits and quick changes |
| Your AI builder's built-in git (Lovable, Bolt, Cursor, Claude Code) | Varies (some paid) | High, inside that tool | Inside your AI builder | When your AI builder already commits, branches, and pushes for you |
| GitHub CLI (`gh`) | Free | Low | Terminal | People comfortable typing commands |

The terms in that first row are the everyday git actions: see [clone](../GLOSSARY.md), [branch](../GLOSSARY.md), [commit](../GLOSSARY.md), and [push](../GLOSSARY.md) for plain-English definitions.

## So is GitHub Desktop required?

No. GitHub Desktop is the most popular beginner-friendly visual option, and it is a fine first choice, but it is not the only one and it is not required. Plenty of teams never install it.

In particular, if your AI builder already pushes to GitHub for you, you may not need a separate "clone it with Desktop" step at all. The tool you are already building in can do the same job.

The one thing that matters is consistency: pick one way and have the whole team use the same one. When everyone follows the same path, it is far easier to help each other when something looks unfamiliar.

For the ideas underneath all of these tools, what a branch is, what a commit is, and why none of it can break anything, read [How Git Works for Non-Coders](../HOW-GIT-WORKS-FOR-NON-CODERS.md).
