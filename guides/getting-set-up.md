# Getting Set Up

> The one-time setup: get your accounts and tools ready, confirm they work, and you're clear to build.

**Goal:** have your accounts and tools ready and confirmed working. **Time:** about 30 to 45 minutes. **Who:** each person, ideally together the first time.

## Why this step is here

We'll be honest with you: more beginners quit during setup than during the actual building. Not because setup is hard, but because small snags feel huge when you're new. A button isn't where the screenshot said it would be, a code gets sent to the wrong email, an app asks for a permission you weren't expecting, and it's easy to decide the whole thing isn't for you.

None of what follows is difficult. It's mostly clicking "sign up" and "yes." But the first time through, it helps to go slowly and to not be alone. So this page walks you through it once, calmly, in order. If you can, do it sitting next to a teammate or with your mentor on a call. Setup goes much faster when someone can glance at your screen and say "oh, click the other one."

For the RecipeRoll team, this was the first thing they all did together: four people on a video call, sharing screens, getting everyone's accounts made in one sitting. By the end of the call, every one of them could open their tools. That's the whole goal of this page.

## What you'll set up

Three things, and that's it:

1. A **GitHub account** — the shared home for your team's project.
2. One **AI builder** — the tool you'll prompt to write the app.
3. A **way to use git** — usually already built into your AI builder, so often there's nothing extra to install.

That's the entire list. Let's do them one at a time.

## Step 1: Create a GitHub account

*[GitHub](../GLOSSARY.md)* is the shared home for your project. It's where your team's code lives so everyone can see it and add to it without overwriting each other's work. You'll have one team project there, and everyone works on the same copy.

To make your account:

1. Go to `github.com`.
2. Click `Sign up` and follow the prompts.
3. Pick a username you wouldn't mind an employer seeing. This is a professional account as much as a project one, so `chef-jordan` is better than something you'll wince at later.
4. When it offers two-factor authentication, turn it on. It sends a second code to your phone when you sign in, which keeps your account safe. It's worth the extra thirty seconds.

That's it. You now have a GitHub account.

## Step 2: Choose and set up an AI builder

Your *[AI builder](../GLOSSARY.md)* is the tool you describe your app to, in plain English, and it does the actual coding. Your team should agree on one so everyone's working in the same place.

To choose, read [the tools overview](../tools/README.md) and the [AI builders comparison](../tools/ai-builders.md). The short version: the browser-based ones like Lovable and Bolt need nothing installed and are the gentlest place to start. You open a web page, you type, and it builds. For a first project, that's usually the right call.

Once your team has picked one:

1. Create your account on its website.
2. Open it and confirm it loads.
3. Type a simple message to it, something like "Hello, can you hear me?" and confirm it responds.

If it answers, you're done with this step. That little reply back is your AI builder saying it's ready.

## Step 3: Get a way to use git

*[Git](../GLOSSARY.md)* is the system that tracks every change to your project and lets a team work on the same code safely. You need a way to use it, but here's the good news: you have two options, and you only need one.

- **Option A — it's already built in.** Most browser AI builders (Lovable, Bolt, and the others) handle git for you behind the scenes. If you're using one of those, you already have a way to use git. There's nothing to install. You can skip the rest of this step.
- **Option B — install GitHub Desktop.** If your tool doesn't handle git, or you'd like a window that shows your project's history plainly, download `GitHub Desktop`, install it, and sign in with the GitHub account you just made.

You do not need both. Pick whichever matches your tool.

> Tip: not sure which option is you? If you chose Lovable or Bolt in Step 2, you're Option A and there's nothing to install.

And to put your mind at ease: you will not have to memorize git commands. The whole point of this setup is that your tools do that work for you. (For more on how the AI does the building, see [how AI building works for non-coders](../HOW-AI-BUILDING-WORKS-FOR-NON-CODERS.md). For a plain-English picture of what git is actually doing, see [how git works for non-coders](../HOW-GIT-WORKS-FOR-NON-CODERS.md).)

## The first 30 minutes: confirm it works

Before you call setup done, run through this quick check. Each one is a small early win, so notice it when it passes:

- You can sign in to GitHub.
- You can see or open your team's project on GitHub.
- Your AI builder responds when you send it a message.
- If you installed GitHub Desktop, it opens and shows your team's project.

If all of those are true, you are genuinely set up. That last box ticked is the moment "I'm going to build software" stops being an idea and becomes a thing you're actually equipped to do.

## Common snags and fixes

If something here trips you up, you're in good company. These are the usual ones, and none of them are a big deal:

- **Sign-in or two-factor trouble.** If a code doesn't arrive or you're locked out, use the recovery options on the sign-in screen, and ask a teammate to walk through it with you. This is the single most common snag, and it's always fixable.
- **"I can't push" or "I don't have access."** This almost always means you haven't been added to the team's *[repository](../GLOSSARY.md)* yet, not that you did anything wrong. Ask whoever created the project to add you as a collaborator, and the error goes away.
- **"Where did my project go?"** When you open a project, it downloads to a folder on your computer. If you've lost track of it, GitHub Desktop shows you exactly where it lives on your machine.
- **Install permission prompts on Mac or Windows.** When you install GitHub Desktop, your computer may ask if you're sure you want to open it, or ask for your password. This is normal and expected. Allow the official app you just downloaded and carry on.

And the catch-all: your AI builder can read any exact error message and explain it in plain English. Copy the whole message, paste it in, and ask what it means and what to try first. If you're still stuck after that, [getting unstuck](getting-unstuck.md) walks you through who to ask and in what order.

## You're set up when

- [ ] Your GitHub account is made.
- [ ] Your AI builder is chosen and responding to messages.
- [ ] You have a way to use git ready (built in, or GitHub Desktop installed).
- [ ] You can see your team's project.
